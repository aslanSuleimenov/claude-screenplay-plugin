# Как начать новый проект

## Шаг 0 — Установи зависимости

```bash
pip install python-docx
```

Это нужно для `/compile` (конвертация в DOCX). Именно `python-docx`, не `docx` — это разные пакеты.

## Шаг 1 — Скопируй папку

```
X:\scenarii\_template     ← не трогаем, это мастер-копия
X:\scenarii\мой_проект    ← сюда копируем всё содержимое
```

**Windows:** при копировании убедись, что папка `.claude/` скопировалась. Это скрытая папка — в Проводнике включи «Показывать скрытые элементы» (Вид → Показать → Скрытые элементы). Без `.claude/` не будут работать команды и хуки.

## Шаг 2 — Открой в Claude Code

```bash
cd "X:/scenarii/мой_проект"
claude
```

(Используй forward slash `/` в путях bash, даже на Windows.)

## Шаг 3 — /startproject

```
/startproject
```

Claude задаст 12 вопросов. **Первый вопрос — тип проекта:**

| Тип | Когда выбирать | Формат страницы |
|-----|----------------|-----------------|
| **Художественный** | Игровой фильм, анимация, сериал, короткометражка | Одноколоночный (screenplay): slug line → действие → имя → диалог |
| **Документальный** | Документальный фильм, корпоративное видео, реклама | Двухколоночный (AV-script): VIDEO ↔ AUDIO таблица |

Остальные вопросы: название, жанр, логлайн, формат, тема, аудитория, место, время, реалии, хронометраж, структура.

Команда заполнит `CLAUDE.md` и настроит формат под выбранный тип.

**Альтернатива:** `python setup.py` — те же вопросы через терминал.

## Шаг 4 — /split (если есть черновик)

```
/split путь/к/черновику.md
```

Прочитает файл и нарежет:
- **Художественный** → сцены с slug lines и диалогами
- **Документальный** → блоки с таблицами VIDEO | AUDIO

Разложит по `scenes/`, вытащит персонажей/героев, заполнит таблицы в `CLAUDE.md`.

Если черновика нет — пропусти, пиши с нуля через `/new-scene`.

## Шаг 5 — /compass

```
/compass жанр — логлайн
```

Залезет в интернет, соберёт жанровый справочник `analytics/compass_artifact.md`.

## Все команды

Команды **автоматически** определяют тип проекта из CLAUDE.md и применяют нужный формат.

### Запуск проекта

| Команда | Что делает |
|---------|------------|
| `/startproject` | Инициализация: тип + 11 вопросов → заполняет CLAUDE.md |
| `/split [файл]` | Режет черновик на сцены/блоки → scenes/ + таблицы |
| `/compass [жанр — логлайн]` | Собирает жанровый справочник из интернета → analytics/ |

### Написание и анализ

| Команда | Худ. фильм | Документальный |
|---------|-----------|---------------|
| `/new-scene 05 Бар` | Создаёт сцену: slug line + действие + диалоги | Создаёт блок: таблица VIDEO ↔ AUDIO |
| `/rewrite 05 замечания` | Переписывает сцену по замечаниям | Переписывает блок по замечаниям |
| `/analyze 05` | Анализ через compass-системы | Анализ через compass-системы |
| `/character-sheet Имя` | Карточка персонажа: арка, речь, противоречия | Карточка героя: появления, цитаты (SOT) |
| `/continuity` | Время, география, реквизит, знания персонажей | Факты, хронология, имена, титры (SUPER) |
| `/research тема` | Веб-исследование → analytics/ | Веб-исследование → analytics/ |

### Структура и управление

| Команда | Что делает |
|---------|------------|
| `/outline` | Показать/создать beat sheet, placeholder-сцены |
| `/renumber` | Перенумеровать сцены (двухфазный rename) |
| `/delete-scene NN` | Удалить сцену + перенумеровать |
| `/merge NN NN` | Объединить две сцены в одну |

### Обзор и сборка

| Команда | Худ. фильм | Документальный |
|---------|-----------|---------------|
| `/stats` | Сцены + хронометраж + диалог/действие | Блоки + V/O слова + SOT кол-во |
| `/compile` | DOCX одноколоночный (screenplay) | DOCX двухколоночный (AV-script, альбомная) |

### Агенты (запускаются вручную)

| Агент | Что делает | Результат |
|-------|------------|-----------|
| `/pitch` | Питч-документ для инвестора/продюсера | analytics/pitch.md |
| `/unico` | Стартовый пакет UNICO (паспорт + библия персонажей + презентация) | analytics/unico_package.md |
| `/proofread [NN|all]` | Корректура: орфография, логика, хронология, анахронизмы | только вывод |

**Когда что использовать:**

| Задача | Инструмент |
|--------|------------|
| Жанровая система, крючок, структура сцены | `/analyze NN` |
| Орфография, логика, хронология, реквизит | `/proofread NN` |
| Полная проверка перед финальной версией | оба подряд |

### Диагностика

| Команда | Что делает |
|---------|------------|
| `/type-check` | Статус проекта: тип установлен? compass есть? сцен сколько? |

### Хуки (работают автоматически)

| Хук | Когда срабатывает | Что проверяет |
|-----|-------------------|--------------|
| Проверка формата | После каждого Edit/Write в scenes/ | **Худ:** slug line, имена, отступы. **Док:** таблица, V/O, SOT, SUPER. **Оба:** AI-паттерны |
| Отчёт сессии | При завершении работы | Записывает изменённые файлы в memory/session_log.md |

## Структура папки

```
мой_проект/
  НАЧНИ_ЗДЕСЬ.md
  CLAUDE.md                       ← заполняет /startproject (включая тип проекта)
  setup.py                        ← альтернатива /startproject через терминал
  scenes/                         ← сцены/блоки (заполняет /split или /new-scene)
  analytics/
    compass_artifact.md           ← создаёт /compass
    avoid-ai-writing-tells.md     ← запрещённые паттерны ИИ-письма
    pitch.md                      ← создаёт агент /pitch
    unico_package.md              ← создаёт агент /unico
  versions/                       ← DOCX (создаёт /compile)
  converter_MD_DOCX/
    md_to_docx.py                 ← конвертер сценария (авто-определяет тип)
    pitch_to_docx.py              ← конвертер питча → DOCX
    README.md                     ← правила форматирования (оба типа)
  memory/
    session_log.md                ← пишет хук автоматически
  .claude/
    settings.json                 ← MCP + хуки (формат-валидация + отчёт)
    commands/                     ← 16 команд (работают с обоими типами)
    hooks/
      session_report.py       ← Python, работает на Windows/macOS/Linux
    agents/                   ← pitch, unico, proofread
```
